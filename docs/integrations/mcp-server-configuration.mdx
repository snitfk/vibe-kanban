<Note>
本页面介绍在 Vibe Kanban **内**为编程代理配置 MCP 服务器。有关将外部 MCP 客户端连接到 Vibe Kanban 的 MCP 服务器，请参阅 [Vibe Kanban MCP 服务器](/integrations/vibe-kanban-mcp-server)指南。
</Note>

## 概述

MCP 服务器通过标准化协议为编程代理提供附加功能。您可以在 Vibe Kanban 中为每个编程代理配置不同的 MCP 服务器，使它们能够访问专门的工具，如浏览器自动化、访问远程日志、通过 Sentry 进行错误跟踪或来自 Notion 的文档。

## 访问 MCP 服务器配置

1. 在 Vibe Kanban 界面中导航到**设置**
2. 在设置菜单中点击 **MCP 服务器**
3. 选择您要为其配置 MCP 服务器的编程代理

## 添加自定义 MCP 服务器

您还可以通过手动配置来添加自己的 MCP 服务器：

<Steps>
<Step title="选择编程代理">
从 MCP 服务器设置页面选择您要为其配置 MCP 服务器的编程代理。
</Step>

<Step title="更新服务器配置 JSON">
在 UI 中显示的服务器配置 JSON 字段中，添加您的自定义 MCP 服务器配置。JSON 将显示所选代理的当前配置，您可以修改它以包含附加服务器。

示例添加：
```json
{
  "mcpServers": {
    "existing_server": {
      "command": "npx",
      "args": ["-y", "some-existing-server"]
    },
    "my_custom_server": {
      "command": "node",
      "args": ["/path/to/my-server.js"]
    }
  }
}
```
</Step>

<Step title="保存并测试">
更新 JSON 配置后：

1. 点击"保存设置"以应用更改
2. 通过使用具有 MCP 功能的代理来测试配置
3. 检查代理日志以查找任何连接问题

<Check>
这些更改会更新编程代理的全局配置文件，即使您停止使用 Vibe Kanban，这些更改也会保留。
</Check>
</Step>
</Steps>

## 最佳实践

<Tip>
**服务器选择**：选择补充编程代理主要功能的 MCP 服务器。例如，对于专注于 Web 开发的代理，使用 Playwright。
</Tip>

<Tip>
**限制 MCP 服务器**：避免向单个编程代理添加过多的 MCP 服务器。过多的服务器和工具会通过大量选项淹没代理，从而降低编程代理的有效性。
</Tip>
