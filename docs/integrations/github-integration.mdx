---
title: "GitHub 集成"
description: "连接到 GitHub 以直接从 Vibe Kanban 创建拉取请求并管理工作流程"
---

Vibe Kanban 与 GitHub 集成，让您可以直接从任务尝试创建拉取请求。此集成依赖于系统上安装并认证的 [GitHub CLI (`gh`)](https://cli.github.com/)。

## 设置

无需在 Vibe Kanban 设置中手动配置 GitHub。集成设计为开箱即用，如果您已安装 GitHub CLI。

### 自动设置
当您尝试创建第一个拉取请求时，Vibe Kanban 将检查 GitHub CLI 是否可用。如果未找到或未认证，您将被引导完成设置过程：

1.  **在 macOS 上**：您将看到一个对话框，提供通过 Homebrew 自动安装 GitHub CLI。
2.  **在 Windows/Linux 上**：您将获得手动安装 CLI 的说明。

### 手动设置
您也可以在使用 Vibe Kanban 之前在终端中手动设置集成：

1.  **安装 GitHub CLI**：按照您操作系统的[官方安装说明](https://github.com/cli/cli#installation)操作。
2.  **认证**：在终端中运行以下命令并按照提示操作：
    ```bash
    gh auth login
    ```
    选择 **GitHub.com**，选择 **HTTPS** 或 **SSH** 作为您的首选协议，并通过 Web 浏览器完成登录。

## 创建拉取请求

GitHub CLI 准备就绪后，您可以直接从任务创建拉取请求：

1.  打开具有您想要合并的更改的任务。
2.  点击**创建 PR**按钮。
3.  将出现一个预填充的对话框：
    *   **标题**：从任务标题派生。
    *   **描述**：从任务描述派生。
    *   **基础分支**：更改的目标分支（默认为仓库的默认分支或尝试中指定的分支）。
4.  点击**创建**在 GitHub 上打开 PR。

如果操作成功，任务状态将更新，并且将提供到新拉取请求的链接。
