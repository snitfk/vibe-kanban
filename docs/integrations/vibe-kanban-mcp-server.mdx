Vibe Kanban 公开一个本地 MCP（模型上下文协议）服务器，允许您从外部 MCP 客户端（如 Claude Desktop、Raycast）甚至在 Vibe Kanban 本身内运行的编程代理管理项目和任务。

<Note>
本页面介绍将**外部 MCP 客户端**连接到 Vibe Kanban 的 MCP 服务器。有关在 Vibe Kanban **内**为编程代理配置 MCP 服务器，请参阅 [MCP 服务器配置](/integrations/mcp-server-configuration)指南。
</Note>

<Info>
Vibe Kanban 的 MCP 服务器是**仅本地**的 - 它在您的计算机上运行，只能由本地安装的应用程序访问。它不能通过公开访问的 URL 访问。
</Info>

## 设置 MCP 集成

### 选项 1：使用 Web 界面

如果您要将 Vibe Kanban MCP 服务器添加到 Vibe Kanban **内**的任何[支持的编程代理](/supported-coding-agents)，这将起作用。

1. 在 Vibe Kanban 设置中，导航到"MCP 服务器"页面
2. 在"流行服务器"部分，点击 Vibe Kanban 卡片
3. 点击 `保存设置` 按钮

<Frame>
<img src="/images/vk-mcp-server-config.jpeg" alt="MCP 服务器配置页面显示如何添加 Vibe Kanban MCP 服务器" />
</Frame>

### 选项 2：手动配置

您可以手动将 MCP 服务器添加到编程代理的配置中。确切的语法将取决于您的编程代理或 MCP 客户端。

将以下配置添加到代理的 MCP 服务器配置中：

```json
{
  "mcpServers": {
    "vibe_kanban": {
      "command": "npx",
      "args": ["-y", "vibe-kanban@latest", "--mcp"]
    }
  }
}
```

## 可用的 MCP 工具

Vibe Kanban MCP 服务器提供以下用于管理项目、任务和任务执行的工具：

### 项目操作

| 工具 | 目的 | 必需参数 | 可选参数 | 返回 |
|------|---------|-------------------|-------------------|---------|
| `list_projects` | 获取所有项目 | 无 | 无 | 带有元数据的项目列表 |

### 任务管理

| 工具 | 目的 | 必需参数 | 可选参数 | 返回 |
|------|---------|-------------------|-------------------|---------|
| `list_tasks` | 列出项目中的任务 | `project_id` | `status`<br/>`limit` | 带有执行状态的任务列表 |
| `create_task` | 创建新任务 | `project_id`<br/>`title` | `description` | 已创建的任务 ID 和确认 |
| `get_task` | 获取任务详情 | `task_id` | 无 | 完整的任务信息 |
| `update_task` | 更新任务详情 | `task_id` | `title`<br/>`description`<br/>`status` | 更新的任务信息 |
| `delete_task` | 删除任务 | `task_id` | 无 | 删除确认 |

### 任务执行

| 工具 | 目的 | 必需参数 | 可选参数 | 返回 |
|------|---------|-------------------|-------------------|---------|
| `start_task_attempt` | 使用编程代理开始处理任务 | `task_id`<br/>`executor`<br/>`base_branch` | `variant` | 尝试 ID 和确认 |

### 支持的执行器

使用 `start_task_attempt` 时，支持以下执行器（不区分大小写，接受连字符或下划线）：

- `claude-code` / `CLAUDE_CODE`
- `amp` / `AMP`
- `gemini` / `GEMINI`
- `codex` / `CODEX`
- `opencode` / `OPENCODE`
- `cursor_agent` / `CURSOR_AGENT`
- `qwen-code` / `QWEN_CODE`
- `copilot` / `COPILOT`
- `droid` / `DROID`

## 使用 MCP 服务器

配置 MCP 服务器后，您可以利用它来简化项目规划和执行工作流程：

1. **规划您的工作**：向 MCP 客户端描述大型功能或项目
2. **请求任务创建**：在任务描述的末尾，只需添加"然后将此计划转换为任务"
3. **自动任务生成**：您的 MCP 客户端将使用 Vibe Kanban MCP 服务器在项目中自动创建结构化任务
4. **开始任务执行**：使用 `start_task_attempt` 以编程方式开始使用特定编程代理处理任务

## 示例用法

### 规划和任务创建

```
我需要构建一个用户认证系统，具有以下功能：
- 带电子邮件验证的用户注册
- 登录/注销功能
- 密码重置功能
- 会话管理
- 受保护的路由

然后将此计划转换为任务。
```

您的 MCP 客户端将使用 `create_task` 工具将其分解为单个任务，并自动将它们添加到您的 Vibe Kanban 项目中。

### 开始任务执行

创建任务后，您可以以编程方式开始处理它们：

```
使用 Claude Code 在 main 分支上开始处理用户注册任务。
```

您的 MCP 客户端将使用 `start_task_attempt` 工具，其参数如下：
```json
{
  "task_id": "123e4567-e89b-12d3-a456-426614174000",
  "executor": "claude-code",
  "base_branch": "main"
}
```

这将创建新的任务尝试，生成功能分支，并在隔离环境中启动编程代理。

### 完整工作流程示例

```
1. 列出所有项目以查找项目 ID
2. 列出项目中的待办任务
3. 为"添加用户个人资料页面"创建新任务
4. 使用 Amp 在 develop 分支上为新任务启动任务尝试
```

每个步骤使用适当的 MCP 工具（`list_projects`、`list_tasks`、`create_task`、`start_task_attempt`）来管理从规划到执行的完整工作流程。

### 内部编程代理（在 Vibe Kanban 内）

一个强大的工作流程涉及使用 Vibe Kanban 内也连接到 Vibe Kanban MCP 服务器的编程代理：

1. **创建规划任务**：使用配置了规划提示的自定义代理配置文件创建任务。有关创建自定义配置文件的详细信息，请参阅[代理配置](/configuration-customisation/agent-configurations)。
2. **探索和规划**：编程代理探索代码库并制定全面的计划
3. **生成任务**：要求编程代理"为此计划创建一系列单独的任务"
4. **自动填充**：代理使用 MCP 服务器直接在 Vibe Kanban 界面中填充单个任务

这创建了一个无缝的工作流程，其中高级规划自动在项目板中生成可操作的任务。
