description: "了解如何审查和提供编程代理所做代码更改的反馈"
---

当编程代理完成任务时，它会自动移至**审查中**列。在这里您可以检查更改，提供反馈，并确保实现符合您的要求。

## 打开代码审查界面

<Steps>
<Step title="访问任务">
  点击**审查中**列中的任何任务以打开它。
</Step>

<Step title="查看差异">
  点击**差异图标**以查看代理所做的所有代码更改。
</Step>
</Steps>

## 添加审查评论

### 特定行评论

对特定代码行提供反馈：

<Steps>
<Step title="定位行">
  在差异视图中找到您想要评论的行。
</Step>

<Step title="添加评论">
  点击行开头的**加号图标** (+) 以创建审查评论。

  <Frame>
  <img src="/images/add-line-comment.png" alt="用于添加行评论的加号图标" />
  </Frame>
</Step>

<Step title="写下您的反馈">
  在出现的文本字段中输入您的评论。您可以提供建议、提出问题或请求更改。
</Step>
</Steps>

### 跨文件的多个评论

您可以在同一审查中跨不同文件创建多个审查评论：

- 在单个文件的多行中添加评论
- 在不同的更改文件之间切换并为每个文件添加评论
- 提交审查时，所有评论将作为完整审查的一部分被收集和发送

<Note>
审查评论不会单独提交。它们会被收集并在您提交反馈时作为完整审查发送。
</Note>

## 提交您的审查

<Steps>
<Step title="提交审查">
  点击**发送**按钮将所有反馈发送给编程代理。

  <Info>
  所有评论都会合并到一条消息中，供编程代理处理。
  </Info>
</Step>

<Step title="任务移回进行中">
  提交后，任务返回到**进行中**列，代理将处理您的反馈并实施请求的更改。
</Step>
</Steps>
