任务尝试代表与编程代理针对任务的单次会话。大多数任务只需要一次尝试，但您可能需要额外的尝试来重新开始。

## 何时创建新任务尝试

在以下情况下创建新任务尝试：

- **从头开始**，在不成功的尝试后使用不同的方法
- **尝试不同的编程代理**（例如，从 Claude 切换到 Codex）
- **使用不同的代理配置文件或变体**以实现专门的行为
- **从不同的基础分支工作**以整合最近的更改
- **重置对话上下文**以完全重新开始

<Tip>
大多数用户每个任务只需要一次尝试。只有在第一种方法不如预期时才创建额外的尝试。
</Tip>

## 创建额外尝试

为现有任务创建新任务尝试：

<Steps>
<Step title="导航到任务">
  打开需要重新尝试的任务。
</Step>

<Step title="点击创建新尝试">
  点击任务右上角的三点图标，然后选择**创建新尝试**。
</Step>

<Step title="配置尝试">
  选择您的代理配置文件、变体和基础分支。这些可以与之前的尝试不同。
</Step>

<Step title="开始执行">
  点击**创建尝试**以使用新配置开始新的执行。
</Step>
</Steps>

## 对子任务的影响

<Warning>
创建新任务尝试会影响子任务。子任务链接到特定的任务尝试，而不是任务本身。
</Warning>

当您创建新任务尝试时：

- **现有子任务**仍然链接到其原始父尝试
- 从新尝试创建的**新子任务**将使用新尝试的分支作为基础

<Info>
有关子任务如何与任务尝试一起工作的更多详细信息，请参阅[创建子任务](/core-features/subtasks)。
</Info>
