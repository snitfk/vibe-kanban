## 创建您的项目
点击**创建项目**按钮从两个选项中选择：
- **从现有 git 仓库**：浏览您的文件系统并从按最近活动排序的 git 仓库列表中选择
- **创建空白项目**：从头开始生成新的 git 仓库
每个项目代表一个 git 仓库。创建后，您可以使用设置脚本、开发服务器脚本和其他设置对其进行配置。


<Note>
创建项目后，您需要按右上角的设置按钮来配置项目脚本和设置。
</Note>

## 项目设置

创建项目后，您可以通过点击右上角的设置按钮访问项目设置。从这里，您可以配置项目的各个方面。

### 设置脚本

设置脚本将在编程代理执行之前运行。这对于安装依赖项很有用，例如您可能运行 `npm install` 或 `cargo build`。这将节省您的时间，因为您的代理不需要弄清楚这些命令尚未运行。

<Note>
每次执行编程代理时，它都在 [git 工作树](https://git-scm.com/docs/git-worktree)中运行，该工作树不太可能包含您的依赖项、配置、.env 等。
</Note>

### 开发服务器脚本

当您从[预览](/core-features/testing-your-application)部分按下"启动开发服务器"按钮时，将运行开发服务器脚本。它对于在编程代理运行后快速审查工作很有用。

### 清理脚本

清理脚本在编程代理完成其回合后运行。您可以使用这些来整理工作区、删除临时文件或执行任何执行后清理。例如，您可能运行 `npm run format` 以确保代码格式正确。将其视为 git pre-commit 钩子。


### 复制文件

要从原始项目目录复制到工作树的文件的逗号分隔列表。这些文件将在创建工作树之后但在设置脚本运行之前复制。对于特定于环境的文件（如 `.env`、配置文件和本地设置）很有用。

<Warning>
确保这些文件被 gitignore，否则它们可能会被提交！
</Warning>
