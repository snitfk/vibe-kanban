当您的任务完成后，Vibe Kanban 提供集成的 git 操作，以保持分支最新并将工作合并回基础分支。

## Git 操作标题

在差异视图顶部，您将看到重要的分支信息和操作：

<Frame>
<img src="/images/vk-diff-header.png" alt="差异标题显示分支信息和操作按钮" />
</Frame>

**分支信息：**
- **任务分支**：您的任务正在工作的分支
- **目标分支**：您将合并到的分支（带有齿轮按钮以更改它）
- **领先提交**：您的分支拥有但目标分支没有的提交数量
- **落后提交**：目标分支中您没有的提交数量（>0 时启用变基按钮）

**操作：**
- **合并**：将您的更改合并到目标分支
- **创建 PR**：在 GitHub 上创建拉取请求
- **变基**：使用目标分支的最新更改更新您的分支

## 变基

点击**变基**以使用目标分支的最新更改更新您的分支。这使您的分支保持最新并维护干净的历史记录。

如果发生冲突，请参阅[解决变基冲突](/core-features/resolving-rebase-conflicts)。

## 合并

点击**合并**将已完成的工作集成到目标分支中。您的任务将自动移至**完成**列，工作树会自动清理。分支会保留，直到您手动删除它。

<Tip>
如果您正在使用 GitHub，请考虑创建拉取请求而不是直接合并。这允许团队审查和 CI 检查。
</Tip>

## 拉取请求管理

### 创建拉取请求

点击**创建 PR**在 GitHub 上创建拉取请求。标题和描述会自动从您的任务详情填充。

<Frame>
<img src="/images/vk-pr-open.png" alt="标题显示创建拉取请求后禁用的推送按钮" />
</Frame>

创建 PR 后，按钮更改为**推送**（在您进行更多更改之前最初被禁用）。

### 更新拉取请求

当您在创建 PR 后继续工作时，**推送**按钮会启用。点击它以将最新更改推送到拉取请求。

<Frame>
<img src="/images/vk-pr-with-push.png" alt="标题显示启用的推送按钮，新更改已准备好" />
</Frame>

当您的 PR 在 GitHub 上合并时，您的任务会自动移至**完成**。
