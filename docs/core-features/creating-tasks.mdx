创建项目后，通过点击项目看板页面右上角的**加号 (+) 图标**添加任务，或使用键盘快捷键 **`c`**。

创建任务时，您有两个选项：

- **创建任务**：将任务添加到看板而不启动编程代理
- **创建并启动**：创建任务并立即使用默认编程代理和当前分支启动它

## 使用任务标签

<Frame>
<img src="/images/screenshot-task-tag-autocomplete.png" alt="任务标签自动完成下拉菜单显示输入 @ 符号后的可用标签" />
</Frame>

添加任务时，您可以使用任务标签插入可重用的文本片段：

1. 在任务描述或后续消息中输入 `@`
2. 开始输入标签名称以过滤可用标签
3. 从下拉列表中选择标签以插入其内容

<Note>
任务标签通过为常见任务结构提供可重用的文本片段来节省时间。在[任务标签](/configuration-customisation/creating-task-tags)指南中了解更多信息。
</Note>

## 启动现有任务

<Frame>
<img src="/images/vk-new-task-attempt.png" alt="任务尝试创建对话框显示代理配置文件和变体选择选项" />
</Frame>

当您打开尚未尝试的任务时，您将看到任务标题、描述和显示"尚无尝试"的尝试列表。点击**加号 (+) 按钮**创建任务尝试并配置：

- **代理配置文件**：从可用代理中选择（例如，CLAUDE_CODE、GEMINI、CODEX）。从设置中预先选择您的默认配置。
- **变体**：如果您选择的代理有变体，请选择适当的变体（例如，DEFAULT、PLAN）。
- **基础分支**：指定代理应从哪个分支工作。默认选择您的当前分支。

<Tip>
使用**创建并启动**将任务添加并立即使用默认设置在一个操作中创建任务尝试。
</Tip>

<Note>
要在任务执行时监控任务，请参阅[监控任务执行](/core-features/monitoring-task-execution)。要了解何时可能需要多次尝试，请参阅[新任务尝试](/core-features/new-task-attempts)。
</Note>

## 通过 MCP 客户端创建任务

<Warning>
这不是创建任务的典型方法，但对于批量任务创建、从其他系统迁移、使用具有额外项目上下文的 AI 助手或想要创建新任务的编程代理非常有价值。
</Warning>

任务也可以使用编程代理或 MCP（模型上下文协议）客户端（如 Claude Desktop 或 Raycast）以编程方式创建。这种方法特别适用于：

- 基于现有数据或项目规范的**批量任务创建**
- 从其他系统（如 Linear、GitHub Issues 或 Jira）**迁移**
- 从项目计划或需求文档**自动生成任务**

<Tip>
有关详细的设置说明和示例，请参阅 [Vibe Kanban MCP 服务器](/integrations/vibe-kanban-mcp-server)文档。
</Tip>

### MCP 任务创建示例

配置 MCP 客户端后，您可以从项目描述创建多个任务：

```
我需要实现用户认证，包括：
- 电子邮件/密码注册
- 带会话管理的登录
- 密码重置功能
- 电子邮件验证
- 受保护的路由中间件

请为每个组件创建单独的任务。
```

MCP 客户端将根据此描述在您的 Vibe Kanban 项目中自动生成结构化任务。

## 理解任务列

任务从"待办"列开始，并根据其进度自动移动：

| 操作 | 列 |
|--------|---------|
| 任务已创建 | 待办 |
| 任务尝试已启动 | 进行中 |
| 任务尝试已完成（成功或失败） | 审查中 |
| 任务尝试已合并 | 完成 |
| PR 在 GitHub 上合并 | 完成 |

<Info>
您可以在列之间手动拖动任务，但这不会触发任何功能。任务移动主要由编程代理操作和 GitHub 集成驱动（每 60 秒轮询一次）。
</Info>
