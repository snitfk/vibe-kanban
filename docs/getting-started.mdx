## 支持的系统

我们在以下系统上构建和测试 Vibe Kanban：

- macOS（Intel 和 Apple Silicon）
- Linux
- Windows

## 前提条件

在安装 Vibe Kanban 之前，请确保您已准备好：

- **Node.js**：推荐使用最新的 LTS 版本
- **编程代理认证**：在 Vibe Kanban 之外完成您首选编程代理的认证

## 安全须知

<Warning>
Vibe Kanban 默认使用 --dangerously-skip-permissions/--yolo 标志运行 AI 代理，这样它们可以自主工作，无需持续的批准提示。每个任务在独立的 git 工作树中运行，防止代理之间互相干扰。
代理仍然可以执行系统级操作，因此请审查它们的工作并保持备份。
</Warning>

## 安装和设置

<Steps>
<Step title="使用编程代理进行认证">
  在启动 Vibe Kanban 之前，请确保您已使用至少一个[支持的编程代理](/supported-coding-agents)进行了认证。按照您首选代理的安装和认证说明操作。
</Step>

<Step title="安装并启动 Vibe Kanban">
  打开终端并运行：

  ```bash
  npx vibe-kanban
  ```

  <Check>
  应用程序将绑定到一个随机的空闲端口，在终端中打印 URL，并自动在默认浏览器中打开。
  </Check>
</Step>

<Step title="完成初始设置">
  完成设置对话框以配置您的编程代理和编辑器偏好设置。GitHub 集成依赖于 GitHub CLI，将在需要时进行配置。
</Step>

<Step title="创建您的第一个项目">
  您将进入项目页面，如果自动发现，该页面将填充您最近活跃的三个 git 项目。点击"创建项目"添加更多项目。
</Step>

<Step title="添加任务">
  通过在项目中[创建任务](/core-features/creating-tasks)开始跟踪您的工作。
</Step>

<Step title="可选：GitHub 集成">
  Vibe Kanban 使用 [GitHub CLI](https://cli.github.com/) 创建拉取请求。确保系统上已安装并认证 `gh`，或在创建第一个拉取请求时按照设置提示操作。
</Step>

<Step title="可选：设置 MCP 集成">
  通过[设置 MCP 集成](/integrations/vibe-kanban-mcp-server)，使用编程代理简化任务创建。
</Step>
</Steps>

<Tip>
要使用固定端口，请指定 `PORT` 环境变量：`PORT=8080 npx vibe-kanban`
</Tip>
